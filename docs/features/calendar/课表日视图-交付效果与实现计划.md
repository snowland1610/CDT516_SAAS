# 课表日视图 - 交付效果与实现计划

> 在现有周视图基础上增加「日视图」：同一页内切换视图模式与日期，复用事件数据与有效期逻辑，单日过滤 + 单日时间条布局。供后续编程实现时对照。

---

## 一、交付效果（可验收）

### 1.1 入口与视图切换

- **入口**：学生端 `/student/calendar`、教师端 `/teacher/calendar` 与当前一致，不新增路由。
- **视图模式**：页面上有「周」与「日」两种模式可切换（如 Tab 或按钮组），默认可为「周」。
- **切换后**：
  - 选「周」：展示当前已有的周视图（一周 7 列 + 时间轴 + 事件块）。
  - 选「日」：展示日视图（单日 + 时间轴 + 该日事件）。

### 1.2 日期状态与导航（日视图下）

- **当前日期**：日视图下有一个「当前选中日期」状态（如 `selectedDate`），默认可为「今天」或当前周视图的周一。
- **导航**：
  - 「上一日」「下一日」：在日视图下切换选中日期。
  - 可选：「今天」按钮，一键回到今天。
  - 可选：简单日期选择（如 input type="date" 或点选日历），直接跳转到某天。
- **日期展示**：在日视图顶部明确展示当前选中的日期（如「2026年2月28日 周五」），与周视图的「本周 2026-02-24 ～ 2026-03-01」区分开。

### 1.3 日视图布局与内容

- **布局**：单列时间轴（与周视图同一时间范围：08:00～18:00，30 分钟一格），左侧时间标签，右侧一列用于该日事件。
- **事件**：只显示「选中日期」当天的排课；事件块样式与周视图一致（课程名、班级、时间、教室、按课程着色、悬停预览、点击弹窗详情）。
- **数据与过滤**：
  - 复用现有 `section_schedule` 拉取逻辑（学生端按 enrollment 的 section，教师端按本人负责课程的 section），**不新增接口**。
  - 从 `scheduleRows` 中过滤出「选中日期的星期几 = 该条 schedule 的 day_of_week」且「选中日期在 [valid_from, valid_to] 内」的事件，再按 start_time 排序展示。
- **空态**：当天无课时，显示「当日暂无课程」或类似提示，时间轴仍保留。

### 1.4 与周视图的一致性

- **事件数据结构**：日视图与周视图共用同一套「事件」结构（如 `WeekEvent` 或抽成 `CalendarEvent`），字段一致：id、courseId、courseName、sectionName、room、start_time、end_time、valid_from、valid_to、教师名（学生端）等。
- **有效期逻辑**：完全复用「选中日期字符串在 [valid_from, valid_to] 内」的判断，与周视图一致。
- **颜色与交互**：使用同一套 `getCourseColorClasses(courseId)`、`eventToGridRow`、悬停 tooltip、点击弹窗详情，视觉与交互与周视图一致。

### 1.5 视图切换时日期联动（建议）

- 周 → 日：可把「当前周的第一天」或「今天」作为日视图的初始选中日期。
- 日 → 周：再切回周时，周视图的 `weekStart` 可保持不变，或根据日视图的 `selectedDate` 所在周更新 `weekStart`，使该日出现在当前周视图中。

## 二、实现计划（供编程依据）

### 2.1 状态设计

| 状态 | 类型 | 说明 |
|------|------|------|
| `viewMode` | `'week' \| 'day'` | 当前视图模式 |
| `weekStart` | `Date` | 周视图下「本周一」日期（已有） |
| `selectedDate` | `Date` | 日视图下「选中的那一天」；切换为日视图时可初始化为今天或 weekStart |

### 2.2 数据流（复用）

- **拉取**：继续用现有 `useEffect` 拉取 `scheduleRows`（学生端 enrollment → section_schedule，教师端 course → section → section_schedule），**不因日视图而增加请求**。
- **周视图**：在现有 `weekEventsByDay` 上不变：按 `weekStart` + `day_of_week` 得到每天日期，再按 `valid_from`/`valid_to` 过滤。
- **日视图**：
  - 由 `selectedDate` 得到「星期几」：`dayOfWeek = selectedDate.getDay() === 0 ? 7 : selectedDate.getDay()`（与后端 1=周一 一致）。
  - 由 `selectedDate` 得到日期字符串：`dateStr = formatDate(selectedDate)`。
  - 过滤：`scheduleRows` 中 `row.day_of_week === dayOfWeek` 且 `dateStr >= row.valid_from && dateStr <= row.valid_to`，且 `row.section?.course` 存在；转为与周视图相同的事件对象，按 `start_time` 排序，得到 `dayEvents`。

### 2.3 日视图 UI 结构

- **顶部**：视图切换（周 / 日）+ 日视图下日期展示与「上一日」「下一日」「今天」。
- **内容区**：
  - `viewMode === 'week'`：渲染现有周视图网格（7 列 + 时间轴 + 网格线 + 事件块）。
  - `viewMode === 'day'`：渲染日视图——单列时间网格（与周视图同一套 `GRID_SLOT_COUNT`、`getSlotLabel`、`eventToGridRow`），仅一列事件块；无课显示空态文案。

### 2.4 可复用与可抽取

| 项 | 说明 |
|----|------|
| `scheduleRows`、fetch 逻辑 | 完全复用 |
| `formatDate`、`getMonday`、星期几换算 | 复用；日视图需「日期 → day_of_week」 |
| `eventToGridRow`、`getCourseColorClasses`、`GRID_SLOT_COUNT`、`getSlotLabel` | 来自 `lib/calendarGrid.ts`、`lib/calendarColors.ts`，日视图继续用 |
| 事件类型（如 `WeekEvent`） | 可重命名为通用 `CalendarEvent` 或在日视图中复用同一类型 |
| 悬停 tooltip、点击弹窗 | 与周视图共用同一套组件或逻辑，入参为事件对象即可 |
| 时间网格线 | 日视图为单列，可沿用「横线 + 竖线」思路，仅一列时竖线可省略或保留一条 |

### 2.5 实现顺序建议

1. **状态与切换**：在日历页增加 `viewMode`、`selectedDate`；增加「周 / 日」切换按钮；日视图下展示 `selectedDate` 与「上一日」「下一日」「今天」。
2. **日视图数据**：在现有 `scheduleRows` 基础上写 `dayEvents` 的 useMemo（按 `selectedDate` → day_of_week + dateStr 过滤 + 排序）。
3. **日视图布局**：单列时间网格（左侧时间标签，右侧一列），用 `eventToGridRow` 放置事件块，样式与周视图事件块一致，复用颜色与交互。
4. **空态与联动**：日视图无课时的提示；视需要做「日视图选中日期」与「周视图 weekStart」的联动（可选）。

### 2.6 文件与范围

- **改动范围**：仅学生端、教师端两个日历页（`app/student/calendar/page.tsx`、`app/teacher/calendar/page.tsx`）；无需改 `lib/calendarGrid.ts`、`lib/calendarColors.ts` 逻辑，仅按需复用。
- **不涉及**：数据库、Supabase 表结构、新接口；不新增路由。
