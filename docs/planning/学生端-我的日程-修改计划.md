# 学生端「我的日程」修改计划

> 将学生端「课表」改名为「我的日程」，新增个人事件、拖拽/拉伸、当日事件板块，并规范事件类型与配色。供后续编程实施使用。

---

## 一、目标与范围

| 项 | 说明 |
|----|------|
| 改名 | 侧栏与页面标题：「课表」→「我的日程」；路由保持 `/student/calendar` |
| 个人事件 | 学生可创建、编辑、删除个人事件，并同步展示在日历（周/月）上 |
| 配色规范 | 上课事件与个人事件采用不同色环/饱和度；用「事件类型枚举 + 配置」固化，便于后续扩展（如小组事件） |
| 交互 | 周视图：个人事件可拖拽改时间/日期、拉伸改时长；月视图：个人事件仅显示为小色块/点，点击弹窗编辑，不拖拽 |
| 当日事件 | 取消日视图；在日历下方新增「当日事件」板块，展示「当前选中日期」的课程+个人事件，按时间排序；选中日期需明显展示 |

---

## 二、事件类型与配色规范

### 2.1 事件类型枚举

| 类型 | 说明 | 数据来源 | 可编辑 | 可拖拽/拉伸 |
|------|------|----------|--------|-------------|
| `course` | 上课事件 | section_schedule | 否 | 否 |
| `personal` | 个人事件 | student_personal_event（新表） | 是 | 是（仅周视图） |
| （预留） | 如 `group` 小组事件 | 后续扩展 | 按需求 | 按需求 |

### 2.2 配色配置（色环 + 饱和度）

- **上课事件（course）**：沿用现有色相，**较高饱和度**（如 200/400 档），与现有 `calendarColors.ts` 的 `COURSE_COLOR_CLASSES` 对齐。
- **个人事件（personal）**：**同一色环不同饱和度**（如 100/300 档或 150/350），与课程色区分且整体协调。
- **扩展**：新类型在配置中增加「色环名 + 饱和度档位」，由统一方法根据 `eventType` 返回 Tailwind 类名。

### 2.3 落地方式

- 新增或扩展 `lib/calendarColors.ts`（或 `lib/calendarEventTypes.ts`）：
  - 定义事件类型常量：`CalendarEventType = 'course' | 'personal'`（及预留类型）。
  - 定义「事件类型 → 色环 + 饱和度」配置（如 `EVENT_TYPE_PALETTE`）。
  - 对外方法：`getEventColorClasses(eventType: CalendarEventType, id?: number): string`  
    - `course`：传入 `courseId`，按 id 取色（保持现有按课程区分）；  
    - `personal`：可固定一两种颜色或按 `id` 取个人事件色环中的一种。
  - 文档注释注明：后续新增类型时在此配置中增加一项即可。

## 三、数据表设计

### 3.1 个人事件表：student_personal_event

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL PRIMARY KEY | 主键 |
| `student_id` | BIGINT NOT NULL REFERENCES student(id) ON DELETE CASCADE | 所属学生 |
| `title` | VARCHAR(200) NOT NULL | 标题 |
| `start_at` | TIMESTAMPTZ NOT NULL | 开始时间（含日期） |
| `end_at` | TIMESTAMPTZ NOT NULL | 结束时间（含日期） |
| `created_at` | TIMESTAMPTZ DEFAULT now() | 创建时间 |
| `updated_at` | TIMESTAMPTZ DEFAULT now() | 更新时间 |

- 约束：`end_at >= start_at`（CHECK 或应用层校验）。
- 索引：`student_id`、`(student_id, start_at)` 便于按学生、按日/周查询。
- 建表 SQL 存放：`supabase/migrations/YYYYMMDD_add_student_personal_event_table.sql`。

## 四、页面与交互

### 4.1 侧栏与标题

- `components/layout/StudentSidebar.tsx`：`label: '课表'` → `label: '我的日程'`。
- `app/student/calendar/page.tsx`：页面标题「我的课表」→「我的日程」。

### 4.2 视图模式

- 移除「日」视图按钮及日视图相关状态/渲染。
- 保留「周」「月」两种视图；增加统一的「当前选中日期」`selectedDate`，用于「当日事件」板块与月视图点击后的联动。

### 4.3 周视图

- **课程事件**：继续从 section_schedule 拉取，按 day_of_week + valid 过滤，用 `course` 配色，不可拖拽/拉伸。
- **个人事件**：从 student_personal_event 按当前周日期范围拉取，与课程事件一起按列（按天）画在时间格子里，用 `personal` 配色。
- **个人事件交互**：
  - **拖拽**：拖动事件块到另一格 → 更新该事件的日期与开始时间（按落点格子计算），并更新 end_at 保持原时长；写回 DB。
  - **拉伸**：拖动事件块上/下边缘 → 仅调整开始或结束时间，写回 DB。
  - 仅个人事件块响应拖拽/拉伸；课程事件不响应。
- 新建入口：周视图或全局提供「新建事件」按钮，打开表单（标题、日期、开始时间、结束时间），提交后写入 student_personal_event 并刷新列表。

### 4.4 月视图

- **课程事件**：沿用现有逻辑，按日统计数量与 courseIds，格子内小色点 +「n 节」。
- **个人事件**：按日统计数量，与课程事件同格展示（如个人事件用不同色点或同格内两种色点）；点击格子弹窗中展示「当日」课程 + 个人事件列表。
- **弹窗**：
  - 课程：只读，展示课程名、时间等。
  - 个人事件：每条可「编辑」「删除」；可提供「新建事件」并预填日期为当前选中日。
- 月视图上**不做**个人事件的拖拽。

### 4.5 「当日事件」板块

- **位置**：日历主体（周视图或月视图）**下方**。
- **数据**：当前「选中日期」`selectedDate` 对应的：
  - 课程：section_schedule 中 day_of_week + valid 匹配该日；
  - 个人事件：student_personal_event 中 start_at/end_at 与该日有交集的记录。
- **展示**：标题区明确展示所选日期，例如「**2025年3月2日（周日）** 当日事件」；列表按开始时间排序，每条一行（时间 + 标题/课程名 + 类型标签）；不画时间轴。
- **选中日期联动**：
  - 周视图：可通过点击列头或增加「选为当日」将某日设为 selectedDate；或默认 selectedDate 为本周某一天（如今天若在本周则今天，否则周一）。
  - 月视图：点击某格打开弹窗时，同时将该格日期设为 selectedDate，「当日事件」随之切换为该日。
- **交互**：列表中个人事件可点击进入编辑/删除；课程仅展示。

## 六、与现有代码的衔接

- **学生端日历页**：`app/student/calendar/page.tsx`，当前为 scheduleRows（section_schedule）、WeekEvent 类型、周/日/月三视图、getCourseColorClasses。
- **配色**：`lib/calendarColors.ts` 当前仅课程色；扩展为事件类型 + 配置后，course 继续用现有色板，personal 用新色板。
- **网格与时间**：`lib/calendarGrid.ts` 的 `eventToGridRow`、`timeToSlotIndex`、`getSlotLabel` 可继续用于个人事件（需将 start_at/end_at 转为「日期当天的 start_time/end_time」参与计算；跨日个人事件可按「当天可见部分」或按开始日展示，按产品约定）。
- **教师端**：本计划仅改学生端；教师端课表仍可为「课表」与现有逻辑，无需同步改名（若需统一再单独改）。

## 七、验收要点

- 侧栏与页面标题为「我的日程」；无日视图，仅周/月。
- 「当日事件」板块始终可见，标题明确显示所选日期，列表为当日课程 + 个人事件按时间排序。
- 个人事件可新建、编辑、删除，并在周/月视图正确显示；个人事件与课程事件配色区分（不同饱和度/色环）。
- 周视图：个人事件可拖拽改日期/时间、拉伸改时长，且 DB 更新正确；课程事件不可拖拽/拉伸。
- 月视图：个人事件为小色块/点，点击弹窗可编辑/删除，不拖拽。
- 事件类型与配色通过配置扩展，便于后续增加小组事件等类型。
