# 学生端组队大厅升级 — 修改计划

> 目标：将组队大厅提升为侧栏独立入口，支持选课后管理组队；课程详情页仅保留「组队信息」只读展示 +「更多操作」跳转。供后续编程实施使用。

---

## 一、需求摘要（已确认）

| 项 | 结论 |
|----|------|
| 组队大厅选课方式 | **A**：进入组队大厅后顶部「选择课程」下拉/列表，选一门课后展示该课组队内容；**且需要 C**：我的课程列表每门课有快捷入口（如「组队」）可直达组队大厅并带上该 courseId |
| 课程详情页「组队信息」粒度 | 只显示**当前学生在该课程下**的组队信息（即学生已选中的 section 的组队情况，只读） |
| 「更多操作」 | 一个按钮放在**组队信息卡片底部**，`title="去组队大厅管理"`（悬浮显示） |

---

## 二、路由与入口约定

| 路径 | 用途 |
|------|------|
| `/student/group-hall` | 组队大厅独立页。支持 query `?courseId=xxx`：有则进入时默认选中该课程并展示其组队内容；无则展示课程选择器，选课后展示 |
| `/student/courses` | 我的课程列表。操作列在「详情」旁增加「组队」链接 → `/student/group-hall?courseId=该课id` |
| `/student/courses/[id]` | 课程详情。原「组队大厅」区块改为「组队信息」（只读），底部「更多操作」→ `/student/group-hall?courseId=[id]` |

---

## 三、修改清单（按实施顺序）

### 步骤 1：侧栏增加「组队大厅」入口

**文件**：`components/layout/StudentSidebar.tsx`

**修改**：在 `items` 中增加一项，建议放在「我的课程」之后：

- `href: '/student/group-hall'`
- `label: '组队大厅'`

**说明**：保持与现有项相同的样式与高亮逻辑（`pathname.startsWith(href)` 等）。

---

### 步骤 2：新增组队大厅页面

**新建文件**：`app/student/group-hall/page.tsx`

**职责**：

1. **读取初始课程**：`useSearchParams().get('courseId')`，得到可选初始 `courseId`。
2. **课程选择器**：页面顶部展示「选择课程」——下拉或卡片列表，数据源为当前学生已选课程（与我的课程列表一致：enrollment → section → course，去重 courseId，展示课程名称等）。选中后更新 URL 或 state（建议用 state + 可选同步到 query，便于分享链接）。
3. **组队内容区**：当有选中的 `courseId` 时，展示该课程的组队大厅内容：
   - 数据范围：该课程下**当前学生已选中的 section**（与现课程详情页一致：enrollment 中 section 属于该 course 的 section 列表）。
   - 按 section 分块：每个 section 内展示「我的小组」（若有）、创建小组（输入+按钮）、可加入的小组列表（加入按钮）、解散/退出按钮等，逻辑与**当前** `app/student/courses/[id]/page.tsx` 中的「组队大厅」区块一致（含 `refreshGroups`、`handleCreate`、`handleJoin`、`handleLeave`、`handleDissolve`）。
4. **空态**：未选课程时，仅展示选择器 + 提示「请选择一门课程查看组队情况」；已选课程但该课下学生无 section 时，提示「您未选读该课程」或「暂无班级」。

**数据与复用**：组队逻辑（拉取 group、group_member、按 section 展示、创建/加入/退出/解散）可从现有课程详情页抽取复用，或在组队大厅页内实现一份（与现有逻辑保持一致：小组 4～6 人、按 section 组队等）。

**URL 与选课联动**：若从「更多操作」或「我的课程·组队」进入且带 `?courseId=xxx`，首屏即选中该课程并展示其组队内容；用户在组队大厅内切换课程时，可更新 `?courseId=yyy` 以便刷新或分享链接。

---

### 步骤 3：课程详情页 —「组队大厅」改为「组队信息」

**文件**：`app/student/courses/[id]/page.tsx`

**修改**：

1. **区块标题与说明**：将「组队大厅」改为「组队信息」，副文案可保留「小组人数 4～6 人，按班级组队」或简化为「当前课程下您的组队情况（仅查看）」。
2. **内容改为只读**：
   - 仍按 **当前学生在该课程下的 section**（即 `mySections`）分块展示。
   - 每个 section 内仅展示：
     - **我的小组**（若有）：小组名称、成员列表（姓名、学号、组长标识），**不提供**「解散」「退出」等操作按钮。
     - **其他小组**：小组名称、成员数（及可选成员名单），**不提供**「加入」按钮。
     - **未组队人数**（可选）：该 section 下未加入任何小组的学生数或「未组队 X 人」。
3. **移除**：创建小组输入框与按钮、加入按钮、解散/退出按钮、以及与之相关的 state（如 `createName`、`actionLoading` 等组队操作相关状态）和事件处理（`handleCreate`、`handleJoin`、`handleLeave`、`handleDissolve`）。若这些逻辑在课程详情页仅用于组队，可删除；若任务详情页等仍依赖，则保留在别处或通过组队大厅页使用。
4. **底部按钮**：在组队信息卡片**底部**增加一个按钮：
   - 文案：`更多操作`
   - `title`（悬浮）：`去组队大厅管理`
   - 链接：`/student/group-hall?courseId=${course.id}`（当前课程 id 即 `id` 或 `course.id`）。

**说明**：组队信息区块仍需要拉取并展示 group、group_member 数据（只读），因此保留对 `groupsBySection`、`myGroupBySection` 等数据的获取与展示，仅去掉所有操作按钮和创建/加入表单。若为简化，可考虑将「组队信息」的只读展示抽成小组件，供课程详情页使用。

---

### 步骤 4：我的课程列表 — 增加「组队」快捷入口（C）

**文件**：`app/student/courses/page.tsx`

**修改**：在操作列中，在现有「详情」链接旁（或同一单元格内）增加「组队」链接：

- 文案：`组队`
- `href`：`/student/group-hall?courseId=${row.courseId}`

**说明**：列表行可能同一 courseId 多行（多 section），每行都带同一 courseId 即可，点击任一行「组队」都进入组队大厅并选中该课程。

---

## 四、数据与依赖（无库表变更）

- 沿用现有表：`section`、`course`、`enrollment`、`group`、`group_member`。
- 组队大厅页与课程详情页的「当前学生在该课程下的 section」逻辑一致：enrollment 中 `student_id = 当前学生` 且 `section.course_id = 当前课程` 的 section 列表。
- 小组人数上限等业务规则与现有一致（如 4～6 人），无需改表。

---

## 五、实施顺序建议

| 顺序 | 步骤 | 说明 |
|------|------|------|
| 1 | 步骤 1：侧栏增加「组队大厅」 | 先有入口，再有点击后的页面 |
| 2 | 步骤 2：新建组队大厅页 | 实现选课 + 组队操作，支持 `?courseId=` |
| 3 | 步骤 3：课程详情「组队大厅」→「组队信息」 | 只读展示 + 底部「更多操作」 |
| 4 | 步骤 4：我的课程列表增加「组队」链接 | 快捷入口 C |

完成 1～4 后，学生可从侧栏进入组队大厅选课管理组队，也可在课程详情查看组队信息后通过「更多操作」跳转，或在我的课程列表直接点「组队」进入对应课程的组队大厅。

---

## 六、验收要点

- 侧栏有「组队大厅」，点击进入 `/student/group-hall`。
- 组队大厅顶部可选择课程；带 `?courseId=xxx` 打开时自动选中该课程并展示组队内容；可创建/加入/退出/解散，行为与原课程详情内组队一致。
- 课程详情页仅显示「组队信息」只读（当前学生在该课程下的 section 组队情况），无创建/加入/退出/解散；底部「更多操作」悬浮显示「去组队大厅管理」，跳转至组队大厅且带上该 courseId。
- 我的课程列表每门课有「组队」链接，跳转至组队大厅并带上该 courseId。
