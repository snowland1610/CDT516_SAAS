## 一、交付范围总览

| 类别 | 交付内容 |

| **数据库** | `announcement` 表建表 SQL（+ 可选 seed），用户复制到 Supabase SQL Editor 执行 |

## 二、数据表设计（announcement）

### 2.1 表结构约定

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGSERIAL PRIMARY KEY | 主键 |
| `title` | VARCHAR(256) NOT NULL | 标题 |
| `body` | TEXT | 正文（纯文本，后续可扩展富文本） |
| `status` | VARCHAR(20) NOT NULL DEFAULT 'published' | 状态：`published` 已发布 / `archived` 已下架 |
| `created_at` | TIMESTAMPTZ DEFAULT now() | 创建时间 |
| `updated_at` | TIMESTAMPTZ DEFAULT now() | 更新时间 |
| `published_at` | TIMESTAMPTZ | 发布时间（可选，若与 created_at 一致可省略；若需「定时发布」可保留） |

- **发布对象**：若本阶段不做「仅教师可见」等，可不建关联表；需要时后续可加 `target_role`（admin/teacher/student）或单独表。
- **RLS**：若 Supabase 项目开启 RLS，需为 `announcement` 配置策略（如 select 对 anon 开放，insert/update 仅 admin；本计划不强制，由用户环境决定）。

### 2.2 建表 SQL 存放位置

- 文件：`supabase/migrations/YYYYMMDD_add_announcement_table.sql`（具体日期由实现时定）。
- 文档说明：在文件头部注释写明「请将本 SQL 复制到 Supabase 项目 → SQL Editor 中执行」。

## 三、路由与页面清单

| 路径 | 角色 | 页面用途 |
|------|------|----------|
| `/admin/announcements` | 教务处 | 公告列表 + 按钮「发布公告」 |
| `/admin/announcements/new` | 教务处 | 发布公告（表单） |
| `/admin/announcements/[id]/edit` | 教务处 | 编辑公告（表单，id 为公告 id） |
| `/admin/announcements/[id]` | 教务处 | 可选：公告详情（或列表行点击弹窗/抽屉；本计划采用「列表行点标题或「查看」进详情页」） |
| `/teacher/announcements` | 教师 | 只读公告列表 |
| `/teacher/announcements/[id]` | 教师 | 只读公告详情 |
| `/student/announcements` | 学生 | 只读公告列表 |
| `/student/announcements/[id]` | 学生 | 只读公告详情 |

**说明**：教务处从列表进「查看」时，可跳转到 `/admin/announcements/[id]` 详情页（只读展示 + 按钮「编辑」）；或列表内直接提供「编辑」入口。本计划约定：教务处有「列表 + 详情 + 编辑」；列表行操作：查看（进详情）、编辑（进 edit 页）；详情页提供「编辑」按钮。

## 四、页面与组件逻辑

### 4.1 教务处：公告列表 `/admin/announcements`

- **布局**：沿用 admin layout（Header + AdminSidebar），主区为本页。
- **标题**：页面主标题「公告管理」；右侧或标题旁按钮「发布公告」→ 跳转 `/admin/announcements/new`。
- **列表**：
  - 数据：`supabase.from('announcement').select('*').order('created_at', { ascending: false })`；可选 `.eq('status', 'published')` 与「已下架」Tab 切换，本阶段可简化为全部列出并在列表中显示 status。
  - 列：标题、发布时间（created_at 或 published_at 格式化）、状态（已发布/已下架）；操作列：「查看」「编辑」；可选「下架」（将 status 改为 archived，仅对已发布生效）。
  - 空态：无数据时展示「暂无公告」，引导「发布公告」。
- **交互**：「查看」→ 跳转 `/admin/announcements/[id]`；「编辑」→ 跳转 `/admin/announcements/[id]/edit`；「下架」→ 调用 update status 后刷新列表。

### 4.2 教务处：发布公告 `/admin/announcements/new`

- **布局**：同上，主区为表单。
- **表单**：
  - 标题：单行输入，必填。
  - 正文：多行文本（textarea），必填。
  - 状态：默认「已发布」；或隐藏（新建即 published）。
- **提交**：`supabase.from('announcement').insert({ title, body, status: 'published' })`；成功后跳转 `/admin/announcements` 或列表并 toast/提示。
- **返回**：链接或按钮「返回公告列表」→ `/admin/announcements`。

### 4.3 教务处：编辑公告 `/admin/announcements/[id]/edit`

- **布局**：同上。
- **数据**：根据 `params.id` 拉取 `announcement` 单条；若不存在或请求失败则提示并返回列表。
- **表单**：同发布，预填 title、body；可修改 status（已发布/已下架）。
- **提交**：`supabase.from('announcement').update({ title, body, status, updated_at: new Date().toISOString() }).eq('id', id)`；成功后跳转列表或详情。
- **返回**：「返回公告列表」或「返回详情」。

### 4.4 教务处：公告详情 `/admin/announcements/[id]`

- **布局**：同上。
- **数据**：根据 `params.id` 拉取单条公告。
- **展示**：标题、正文（白空格保留或简单换行）、发布时间、状态；按钮「编辑」→ `/admin/announcements/[id]/edit`。
- **返回**：「返回公告列表」。

### 4.5 教师端：公告列表 `/teacher/announcements`

- **数据**：`supabase.from('announcement').select('id, title, created_at').order('created_at', { ascending: false })`；仅展示已发布（`.eq('status', 'published')`），或本阶段全部展示由后端约束。
- **列表**：表格或卡片：标题、发布时间；点击行或「查看」→ 跳转 `/teacher/announcements/[id]`。
- **空态**：「暂无公告」。

### 4.6 教师端：公告详情 `/teacher/announcements/[id]`

- **数据**：按 id 拉取单条；仅展示，无编辑。
- **展示**：标题、正文、发布时间；返回「公告列表」。

### 4.7 学生端：公告列表 `/student/announcements`

- **逻辑**：与教师端一致，只读列表；点击进详情。

### 4.8 学生端：公告详情 `/student/announcements/[id]`

- **逻辑**：与教师端一致，只读详情。

### 4.9 三端工作台「近期公告」

- **教务处** `app/admin/page.tsx`：从 `announcement` 表取最新 3～5 条（如 `.order('created_at', { ascending: false }).limit(5)`），仅 `status = 'published'`；展示标题 + 时间，点击标题或「查看」进 `/admin/announcements/[id]` 或列表。
- **教师** `app/teacher/page.tsx`：同上，展示最新几条，点击进 `/teacher/announcements/[id]`。
- **学生** `app/student/page.tsx`：同上，进 `/student/announcements/[id]`。

若列表为空，展示「暂无公告」（与当前占位一致，去掉「阶段 6 将接入」的说明）。

## 五、组件与复用

- **列表展示**：教务处列表与教师/学生列表可复用「列表行」的展示逻辑（标题、时间），但教务处多「操作」列与「状态」列；可抽成共用组件（如 `AnnouncementRow`）或各页自写，视实现复杂度决定。
- **详情展示**：教师/学生详情可复用同一展示结构（标题、正文、时间）；教务处详情在此基础上增加「编辑」按钮与状态展示。
- **表单**：发布与编辑表单字段一致，可抽成共用表单组件（受控组件，接收 value + onChange 或 defaultValues），发布页与编辑页分别传入初始值与提交回调。

## 六、实现步骤（推荐顺序）

| 步骤 | 内容 | 产出 |
|------|------|------|
| **6.0** | 编写 `announcement` 表建表 SQL，放入 `supabase/migrations/`，并在文件内说明「请复制到 Supabase SQL Editor 执行」；可选：写入 1～2 条 seed 便于联调 | 建表 SQL；用户执行后云端有表 |
| **6.1** | 教务处公告列表页：拉取列表、展示表格、空态、「发布公告」按钮、操作「查看」「编辑」；下架可选 | `app/admin/announcements/page.tsx` |
| **6.2** | 教务处发布公告页：表单（标题、正文）、提交写入 `announcement`、成功后跳列表 | `app/admin/announcements/new/page.tsx` |
| **6.3** | 教务处编辑公告页：根据 id 拉取数据、表单同发布、提交 update、成功后跳列表或详情 | `app/admin/announcements/[id]/edit/page.tsx` |
| **6.4** | 教务处公告详情页：根据 id 拉取、展示标题/正文/时间/状态、「编辑」按钮、返回列表 | `app/admin/announcements/[id]/page.tsx`（与 edit 同目录层级，[id] 为详情，[id]/edit 为编辑） |
| **6.5** | 教师端公告列表 + 详情：只读列表、只读详情 | `app/teacher/announcements/page.tsx`、`app/teacher/announcements/[id]/page.tsx` |

## 七、与现有代码的衔接

- **Supabase 客户端**：全站统一使用 `lib/supabase.ts` 的 `createClient()`。
- **Auth**：无需按用户过滤公告（公告对三端可见）；若后续做「仅教师可见」等，再在查询或 RLS 中加条件。
- **样式**：沿用现有设计基线（`rounded-card`、`shadow-card`、`primary` 按钮、表格与卡片风格），与阶段 3/4 UI 一致。
- **路由守卫**：admin/teacher/student 的 layout 已存在，无需新增；仅新增各端下的 announcements 子路由即可。

## 八、验收标准

- 教务处在 Supabase 执行建表 SQL 后，可发布、编辑、下架公告，并在列表与详情中正确展示。
- 教师、学生侧栏进入「教务处公告」可看到只读列表与详情。
- 三端工作台「近期公告」展示最新若干条公告，点击可进入对应端公告详情或列表。
- 空态、加载态、错误态（如 id 不存在）有合理提示，不崩溃。
