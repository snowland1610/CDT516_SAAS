# 高校教务协作 SAAS — 页面设计需求文档

> 本文档约定各页面结构、内容与路由，作为后续生成前端页面的需求依据。登录方式：选身份 + 选用户，无需密码。

---

## 一、技术选型与约定

### 1.1 推荐技术栈

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | **Next.js 14（App Router）** | 单应用内三端路由、服务端能力便于后续扩展、部署简单 |
| 样式 | **Tailwind CSS** | 原子类、易维护、与组件库搭配好 |
| 组件库 | **shadcn/ui** | 基于 Radix，样式可改、界面简洁现代，符合「美观简洁」 |
| 数据与请求 | **Supabase Client（@supabase/supabase-js）** | 直连现有 Supabase 库，RLS 可选 |
| 状态 | **React Context + 本地 state** | 当前用户（选中的身份与用户）存 Context，页面级用 state |

选型理由：Next.js 单仓库即可覆盖三端、Tailwind + shadcn 易做出统一简洁的界面，且与 Supabase 官方推荐一致。

### 1.2 登录与权限约定

- **登录方式**：选身份 + 选用户，无需密码。
- **流程**：主页面选择角色（教务处 / 教师 / 学生）→ 进入该角色的「选用户」页 → 选择具体账号（如教师选张老师/李老师，学生选学生01～学生50）→ 写入「当前用户」状态并跳转该角色工作台。
- **登出**：清除当前用户状态，回到主页面。
- **路由守卫**：未选用户时访问各端子页面，重定向回主页面或选用户页。

### 1.3 全局约定

- **布局**：各端共用顶部导航栏（含 Logo/应用名、当前角色与用户名、登出）；左侧或顶部为当前端内导航（工作台、课程、公告等）。
- **风格**：简洁、留白适中、信息层级清晰；主色与强调色统一，避免花哨动效。
- **响应式**：优先保证桌面端（1280px 及以上）演示效果；移动端可做基础适配。

## 二、主页面（入口）

### 2.1 路由与用途

- **路径**：`/`
- **用途**：未登录时的唯一入口，提供三个登录入口，无密码。

### 2.2 页面结构

- **顶部**：应用名称（如「高校教务协作平台」）、可选副标题。
- **主体**：三个入口卡片（或大按钮），横向或网格排列：
  1. **教务处**：图标 + 文案「教务处管理」+ 简短说明（如发布公告、查看课程与统计）。
  2. **教师**：图标 + 文案「教师端」+ 简短说明（如管理课程、发布任务、批改作业）。
  3. **学生**：图标 + 文案「学生端」+ 简短说明（如我的课程、组队、提交作业、查成绩）。
- **交互**：点击某一入口 → 跳转到该角色的「选用户」页（见下）。

### 2.3 内容与文案

| 区域 | 内容 |
|------|------|
| 标题 | 高校教务协作平台 |
| 教务处入口 | 教务处管理：发布公告、管理课程与统计 |
| 教师入口 | 教师端：课程与任务、批改与成绩 |
| 学生入口 | 学生端：我的课程、组队与提交、成绩查看 |

## 三、选用户页（按角色）

### 3.1 路由与用途

- **路径**：`/select-user?role=admin` | `/select-user?role=teacher` | `/select-user?role=student`
- **用途**：选定角色后，选择要「扮演」的具体用户；选完即视为登录，写入全局状态并跳转该角色工作台。

### 3.2 页面结构

- **顶部**：返回首页链接；当前角色标签（教务处 / 教师 / 学生）。
- **主体**：
  - **教务处**：仅 1 个用户「教务处管理员」，点击即进入教务处工作台。
  - **教师**：列表展示 2 名教师（张老师、李老师），显示姓名/工号，点击某一项即进入教师工作台。
  - **学生**：列表展示 50 名学生（学生01～学生50），可分页或滚动；每项显示姓名、学号；点击某一项即进入学生工作台。
- **交互**：选择后写入「当前用户」（含 id、role、display_name 等），跳转至对应该角色的工作台路径。

### 3.3 内容与数据来源

| 角色 | 展示列表来源 | 展示字段 |
|------|--------------|----------|
| 教务处 | 固定 1 条或从 `user` 表查 role=admin | display_name |
| 教师 | `user` + `teacher`，role=teacher | name, employee_no |
| 学生 | `user` + `student`，role=student | name, student_no |

## 四、教务处端页面

### 4.1 路由前缀与布局

- **前缀**：`/admin`
- **布局**：顶部栏（应用名 + 当前「教务处管理员」+ 登出）；侧栏：工作台、课程目录、公告、统计（可选）。

### 4.2 工作台 `/admin`

| 区块 | 内容 |
|------|------|
| 标题 | 工作台 / 教务处首页 |
| 统计卡片（可选） | 课程总数、公告总数、可选：待办数量 |
| 近期公告 | 最新 3～5 条公告（标题、发布时间），点击进公告详情或列表 |
| 快捷入口 | 进入「课程目录」、进入「发布公告」 |

若暂无公告表，可先做静态占位或「暂无公告」提示。

### 4.3 课程目录 `/admin/courses`

| 区块 | 内容 |
|------|------|
| 标题 | 课程目录 |
| 列表 | 表格或卡片：课程名称、所属专业、负责教师、班级数（section 数）、选课人数（enrollment 汇总）；每行可点进「课程详情」 |
| 数据 | 从 `course` 联表 `major`、`teacher`，并统计 `section`、`enrollment` |

### 4.4 课程详情 `/admin/courses/[id]`

| 区块 | 内容 |
|------|------|
| 标题 | 课程名称 + 返回课程目录 |
| 基本信息 | 课程名、专业、负责教师、课程代码 |
| 班级列表 | 该课程下 section 列表（section1、section2…），每班选课人数；可选：课表（从 section_schedule 读） |
| 可选 | 该课程下学生名单（通过 enrollment 反查 student） |

### 4.5 公告列表 `/admin/announcements`

| 区块 | 内容 |
|------|------|
| 标题 | 公告管理 + 按钮「发布公告」 |
| 列表 | 表格：标题、发布时间、状态（已发布/已下架）、操作（查看、编辑、下架） |
| 数据 | 若已有公告表则从表读；若无则先做静态占位或空状态 |

### 4.6 发布/编辑公告 `/admin/announcements/new`、`/admin/announcements/[id]/edit`

| 区块 | 内容 |
|------|------|
| 表单 | 标题、正文（富文本或纯文本）、发布对象（全校/教师/学生）、发布时间；提交后回到公告列表 |

（若第一阶段不做公告表，可仅做页面骨架与表单，提交暂不落库。）

### 4.7 统计 `/admin/stats`（可选）

| 区块 | 内容 |
|------|------|
| 标题 | 课程与任务统计 |
| 内容 | 按课程汇总：任务数、提交率、平均分等；可选：公告阅读情况。数据来自 task、submission 等表（需建表后实现） |

## 五、教师端页面

### 5.1 路由前缀与布局

- **前缀**：`/teacher`
- **布局**：顶部栏（应用名 + 当前教师姓名 + 登出）；侧栏：工作台、我的课程、公告（查看教务处公告）。

### 5.2 工作台 `/teacher`

| 区块 | 内容 |
|------|------|
| 标题 | 工作台 |
| 统计卡片 | 我的课程数、待批改任务数（若已有 task/submission 表） |
| 近期待办 | 即将截止的任务、未批改提交数（可选） |
| 教务处公告 | 最新 3～5 条公告，点击进公告详情 |
| 快捷入口 | 进入「我的课程」 |

### 5.3 我的课程 `/teacher/courses`

| 区块 | 内容 |
|------|------|
| 标题 | 我的课程 |
| 列表 | 当前教师负责的课程卡片/列表：课程名称、班级数、选课人数；点击进入该课程详情 |
| 数据 | 从 `course` 查 teacher_id = 当前教师 id |

### 5.4 课程详情 `/teacher/courses/[id]`

| 区块 | 内容 |
|------|------|
| 标题 | 课程名称 + 返回我的课程 |
| 基本信息 | 课程名、课程代码、简介（若有） |
| Tab 或分块 | **班级与课表**：section 列表 + 每班课表（section_schedule）；**学生名单**：该课程下所有选课学生（通过 enrollment 关联 section → course）；**任务列表**：该课程下任务（需 task 表），状态、截止时间、提交数 |
| 操作 | 按钮「发布任务」→ 跳转发布任务页 |

### 5.5 发布任务 `/teacher/courses/[id]/tasks/new`

| 区块 | 内容 |
|------|------|
| 表单 | 任务标题、任务说明、截止时间、是否允许组队、小组人数范围（最小/最大）、组队截止时间；可选：附件上传 |
| 提交 | 提交后创建 task 记录，回到课程详情任务列表 |

（需 `task` 表；若尚未建表，先做页面与表单，提交逻辑后补。）

### 5.6 任务详情（教师视角） `/teacher/courses/[courseId]/tasks/[taskId]`

| 区块 | 内容 |
|------|------|
| 标题 | 任务名称 + 返回课程详情 |
| 任务信息 | 标题、说明、截止时间、组队规则、组队截止时间 |
| 提交列表 | 已提交的小组/名单、提交时间、附件或链接；每行操作「批改」 |
| 统计 | 提交率、未提交小组数（需 group、submission 表） |

### 5.7 批改页 `/teacher/courses/[courseId]/tasks/[taskId]/grade/[submissionId]`

| 区块 | 内容 |
|------|------|
| 标题 | 批改：任务名 + 小组/提交标识 |
| 提交内容 | 展示学生提交的文本/链接/附件 |
| 表单 | 分数、评语；按钮「保存」「发布成绩」 |
| 数据 | 更新 submission 表，发布后学生端可见成绩 |

### 5.8 公告列表（教师） `/teacher/announcements`

| 区块 | 内容 |
|------|------|
| 标题 | 教务处公告 |
| 列表 | 只读列表：标题、发布时间；点击进详情 |
| 详情 | 标题、正文、发布时间 |

## 六、学生端页面

### 6.1 路由前缀与布局

- **前缀**：`/student`
- **布局**：顶部栏（应用名 + 当前学生姓名/学号 + 登出）；侧栏：工作台、我的课程、公告。

### 6.2 工作台 `/student`

| 区块 | 内容 |
|------|------|
| 标题 | 工作台 |
| 我的课程 | 当前学生已选课程列表（通过 enrollment 查 section → course），每项可进课程详情 |
| 待办提醒 | 待完成任务（未提交且未过期）、待组队提醒（需组队且未加入/未建组）；可选 |
| 成绩与反馈 | 最近批改成绩摘要（需 submission 表）；可选 |
| 教务处公告 | 最新 3～5 条，点击进详情 |

### 6.3 我的课程 `/student/courses`

| 区块 | 内容 |
|------|------|
| 标题 | 我的课程 |
| 列表 | 当前学生选中的课程（含所在班级 section）；每项显示课程名、班级名、教师；点击进课程详情 |
| 数据 | enrollment → section → course，filter student_id = 当前学生 |

### 6.4 课程详情（学生） `/student/courses/[id]`

此处 `id` 为 **course.id**；若学生在该课程下有多个 section，可默认选第一个或由前端选一个 section 上下文。

| 区块 | 内容 |
|------|------|
| 标题 | 课程名称 + 返回我的课程 |
| 课程信息 | 课程名、教师、当前学生所在班级（section） |
| 课表 | 该 section 的 section_schedule，展示星期、时段、教室、有效期 |
| 任务列表 | 该课程下任务；每项显示标题、截止时间、是否需组队、本人/本组提交状态（已提交/已批改/未提交） |
| 操作 | 点击任务 → 进入任务详情（学生） |

### 6.5 任务详情（学生） `/student/courses/[courseId]/tasks/[taskId]`

| 区块 | 内容 |
|------|------|
| 标题 | 任务名称 + 返回课程详情 |
| 任务说明 | 标题、说明、截止时间、组队规则（人数范围、组队截止时间） |
| 我的小组 | 若已组队：小组名、成员列表、组长；操作「退出小组」（若允许）、「提交作业」；若未组队：按钮「创建小组」或「申请加入小组」 |
| 组队区 | 未组队时：创建小组（小组名）；或展示可加入的小组列表，申请加入（需 group、group_member 表） |
| 提交区 | 已组队且未提交：提交表单（文本/链接/附件）；已提交：提交记录、时间；已发布成绩：分数、评语 |

### 6.6 公告列表（学生） `/student/announcements`

| 区块 | 内容 |
|------|------|
| 同教师端 | 只读教务处公告列表 + 详情，无编辑能力 |

## 八、数据依赖与建表说明

- **已有表**：`major`、`user`、`teacher`、`student`、`course`、`section`、`section_schedule`、`enrollment`。可直接支撑：主页面、选用户、三端工作台、课程目录与课程详情、课表、学生名单（通过 enrollment）。
- **公告**：若需公告列表与发布，需新增 `announcement` 表（标题、正文、发布对象、发布时间、状态等）；本文档已预留页面，表结构可在实现时补充。
- **任务与小组**：需新增 `task`、`group`、`group_member`、`submission` 表后，方可完整实现：教师发布任务、学生组队、提交、教师批改与成绩发布。实现这些页面时再补充建表 SQL 与 seed（或通过界面操作产生数据）。

## 九、需要你决定的项

- **应用名称**：当前文档使用「高校教务协作平台」，如需改名可在此定稿，后续页面标题与主页面统一使用该名称。
- **公告功能优先级**：若希望演示「教务处发公告 → 教师/学生看公告」，需在实现前确定是否新增 `announcement` 表并写入建表与 seed；若暂不演示，可只做公告列表/详情的空状态或占位。
- **任务/小组/提交的落地顺序**：建议先做「教师发任务 + 学生看任务 + 组队 + 提交 + 教师批改」这条链，再补教务处统计等；若你希望先做纯课程与课表、后做任务，也可说明，我会按此顺序生成页面。

除此三点外，其余按本文档执行即可；后续生成页面时将以本文档为需求依据。
