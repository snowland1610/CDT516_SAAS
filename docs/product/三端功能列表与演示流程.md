# 高校教务协作 SAAS — 三端功能列表与演示流程

> 围绕「教务处 — 教师 — 学生」信息打通 + 小组组队 + 小组任务管理，按演示流程组织的功能清单。

---

## 一、演示流程总览

演示建议按「教务处 → 教师 → 学生」的真实使用顺序进行，体现信息自上而下流通；再通过学生的组队与任务提交，体现自下而上的反馈与闭环。

| 阶段 | 角色 | 演示重点 |
|------|------|----------|
| 1 | 教务处 | 发布学期/课程基础数据、公告，让教师和学生有「可用的环境」 |
| 2 | 教师 | 创建/认领课程、发布任务、设置组队与提交规则 |
| 3 | 学生 | 查看课程与任务、完成组队、提交作业、查看成绩与反馈 |
| 4 | 教师 | 批改、打分、反馈；可选：查看统计 |
| 5 | 教务处 | 查看整体统计与完成情况（可选） |

## 五、按演示顺序的推荐功能路径

便于演示时「按顺序点击」不遗漏核心功能：

1. **教务处**：登录 → 创建/选择学期 → 发布一条公告 →（可选）查看课程列表。
2. **教师**：登录 → 查看公告 → 创建/进入一门课程 → 发布一个任务并设置组队规则（如 3–5 人、组队截止时间）→ 查看学生名单。
3. **学生 A**：登录 → 查看公告 → 进入课程 → 查看任务 → 创建小组。
4. **学生 B、C**：登录 → 进入同一课程与任务 → 申请加入学生 A 的小组；学生 A 通过。
5. **学生（组长）**：在任务下以小组身份提交作业。
6. **教师**：进入该任务 → 查看提交列表 → 批改、打分、写评语 → 发布成绩。
7. **学生**：查看成绩与评语。
8. **教务处（可选）**：登录 → 查看课程任务统计或公告阅读情况。
